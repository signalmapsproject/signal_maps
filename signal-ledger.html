Signal Ledger.
Chronology with uncertainty attached.

This page is the project’s working ledger: a timeline-first index of visible events, delayed signals, proxy indicators, disputed claims, and unresolved questions. It is built to show not only what happened, but when it became visible, how the signal moved, and what remains unsettled.

Each row distinguishes between event time and signal time, because delay, routing, and asymmetry are part of the evidence structure rather than background noise. The result is a chronology that keeps facts, inferences, disputes, and narrative pressure in the same frame without treating them as equivalent.

Structure

How to read the ledger

This page works like a timeline with evidence tags. A row can log an event, a later attribution shift, a proxy leak, or a framing signal. Rows do not all carry the same evidentiary weight, and they are not meant to.

Primary sort
Date / Time

Newest signal first by default, with event time preserved inside each row.

Split timing
2 clocks

Event time and signal time stay separate so delays remain visible.

Method
Routing

Where a signal appeared first matters almost as much as the claim itself.

Status
Open / Contested / Resolved

Rows track whether a question is settled, still live, or disputed.

Column logic

Core schema

  • Event date / time logs when the underlying event allegedly happened.
  • Signal date / time logs when the claim or evidence entered the visible record.
  • Epistemic type distinguishes verified facts, proxy indicators, probable inferences, disputed claims, and narrative signals.
  • Routing note records where a signal surfaced first, where it lagged, and where it stayed missing.
  • What would resolve it keeps unresolved questions tied to observable next checks instead of vague uncertainty.
Reading note

What this page is for

  • The ledger is a working public record, not a claim that every row has equal evidentiary weight.
  • It keeps verified facts, proxy indicators, disputed claims, and interpretive signals in one place while preserving the differences between them.
  • That makes it easier to track how visibility, confidence, and meaning shift over time.
